Sample deliverable · Product-update enablement

One release note, three formats, one day

When a product ships fast, enablement’s job is to turn each release note into the right learning for each audience before it surfaces as a wave of tickets. This sample takes a real Harvey release and shows the same change rendered three ways — a team blast, a specialist quick-update, and a knowledge-base article — plus how I’d decide what goes where and how I’d know it landed.

What’s New

Harvey is now available inside Microsoft 365 Copilot. You can call @Harvey in a Copilot chat or open the Harvey agent from the sidebar to ask legal questions, run research, analyze documents, and pull in Harvey Vault content — without leaving Microsoft 365. It also works with Copilot Cowork, Microsoft’s agentic assistant: delegate a goal and let it plan and run multi-step work across email, calendar, files, and Teams, drawing on Harvey’s legal intelligence along the way.

Why It Matters

It brings Harvey’s legal depth into the tools people already work in, cutting the tool-switching and manual context-gathering. When a quick question turns into substantive work, you can open the thread in Harvey’s full environment for deeper reasoning and drafting — with the Copilot conversation preserved in context.

How to Use

Admin opt-in, US to start. Setup is covered in Harvey’s Help Center article on the integration.

Format 2

Specialist quick-update — enough to handle the tickets

1 What changed

Harvey now runs inside Microsoft 365 Copilot: call @Harvey in a Copilot chat or open the sidebar agent to ask questions, analyze documents, and pull Harvey Vault content. It also works with Copilot Cowork for delegated, multi-step work across email, calendar, files, and Teams.

2 Who’ll ask, and what to say

“Is this on for us?” — It’s admin opt-in and US-only to start; check with your workspace admin.

“Copilot or the full Harvey app?” — Copilot for quick questions and analysis in your flow; when it becomes substantive, multi-step work, open the thread in full Harvey — your Copilot conversation carries over.

“Can it use our Vault?” — Yes, it can pull Harvey Vault content, subject to the same permissions.

Unchanged: answers are still cited — coach users to verify against the source before relying on the output.

3 If something looks off

“I don’t see Harvey in Copilot.” Confirm the admin has opted in and the user is in a supported region (US). Vault content missing? Check Vault permissions and source scope, same as in-app. Route substantive accuracy issues through normal escalation and tag them, so we can see whether this new surface changes ticket patterns.

Format 3

Knowledge-base article — the durable, linkable answer

Using Harvey in Microsoft 365 Copilot

What it is: call @Harvey in a Copilot chat or open the sidebar agent to ask legal questions, analyze documents, and pull Harvey Vault content inside Microsoft 365. It also works with Copilot Cowork for multi-step tasks across your Microsoft apps.

Copilot vs. the full Harvey app: use Copilot for quick, in-flow questions and analysis; move to full Harvey for substantive, multi-step work — open the thread there and your Copilot conversation is preserved in context.

Good to know: this is admin opt-in and available in the US to start, and answers are cited — expand a citation and confirm it against the source before you rely on it.

If you don’t see it: confirm your admin has enabled the integration and that you’re in a supported region; full setup steps are in Harvey’s Help Center article on the integration.

Closing the loop

How I’d know it landed

Content isn’t done when it ships — it’s done when it changes something. My primary metric for a release like this is the topic ticket rate: tickets about “enabling Harvey in Copilot” and “Copilot vs. Harvey” per 100 active users, tracked week over week. If awareness landed, it shouldn’t spike at rollout; if it climbs, the blast or the article missed. Around it I’d watch two supporting signals — the KB article’s “was this helpful?” rate (is the content clear?) and whether tagged accuracy tickets shift on the new Copilot surface (a new failure mode to retrain on?).

One distinction I care about: ticket rate is a lagging signal, so I pair it with a leading one — specialist confidence on the topic, sampled in a 30-second pulse check — to catch confusion before it reaches the queue. Those signals decide what to revise, which is the start of the next update, not the end of this one.
