Specialist job aid: triaging a citation-accuracy ticket

A one-page quick-reference for a support specialist who receives a ticket that says, in effect, “Harvey’s answer cited something that doesn’t look right.” The goal is a fast, consistent triage path, a clear escalation line, and a calm, accurate message back to the user.

Orientation

Why this ticket matters

Citations are how Harvey lets a user verify an answer before they rely on it — the ability to check a claim against its source is core to trust in the product. So a citation that looks wrong isn’t a cosmetic bug to the user; it’s a question about whether they can trust the tool at all. How a specialist handles it shapes confidence well beyond the single ticket.

First, classify the report in one read

“Citation accuracy” usually means one of four things. Naming which one you’re looking at decides everything that follows: a wrong link (citation opens the wrong place), a mismatch (the cited source doesn’t support the sentence), a missing source (a claim with no citation), or a scope issue (the user expected a source Harvey wasn’t given access to).

Triage

The 5-step path

Work top to bottom. Most tickets resolve by step 3.

  1. Reproduce the scope. Confirm what sources the query actually ran over — uploads, a Vault, a DMS connection, premium databases, or web. Most “wrong” citations are really a source-scope mismatch: Harvey can only cite what it was given.
  2. Open the citation. Click through to the cited passage and read it against the sentence it supports. Decide which of the four categories (above) you’re in.
  3. Check for a known pattern. Long, multi-source documents and look-alike versions are the usual culprits. Confirm the user is on the current version of the document set and that the source file isn’t a near-duplicate.
  4. Coach the verification habit. Show the user how to expand a citation and confirm a claim at the source before relying on it — the single most valuable habit for trustworthy output. This both resolves the moment and prevents the next ticket.
  5. Decide: resolve or escalate. If it’s scope, version, or how-to — resolve and document. If the source genuinely does not support the claim, it’s a substantive accuracy issue — escalate (criteria below).

Decision line

Escalate when…

01

The source truly doesn’t support the claim

A reproducible mismatch between an answer and its cited source. Attach query, answer, and the cited passage.

02

It repeats across queries

The same error pattern shows up on more than one prompt or user — a signal, not a one-off.

03

It touches data scope or security

Anything suggesting the answer reached, or missed, sources in a way that implicates permissions or confidentiality — escalate immediately.

04

The user’s trust is shaken

Even if minor, if a key account is rattled, loop in the lead so the relationship is handled, not just the ticket.

Always log the category

Tag every citation ticket with one of the four categories before you close it. That tag is what turns scattered tickets into a trend the enablement and QA teams can act on — and it’s how a recurring mismatch becomes the next training module instead of a hundred separate conversations.

Adjacent ticket

When it’s a data question, not an accuracy one

Citation tickets often shade into “…and wait, where is our data going?” That’s a different class of question. You don’t resolve it with the triage above — you answer the headline confidently and route the specifics.

01

Do you train on our data?

No — Harvey contractually guarantees it doesn’t use your inputs, outputs, or uploads to train the underlying models.

02

Where is our data stored?

In-region (EU/Switzerland, US, or Australia). Customers control what’s uploaded, retention, and deletion.

03

Who can see it?

Access controls (SAML SSO, audit logs, IP allow-listing) and in-product permissions govern who can access, view, and edit content.

04

Is it independently checked?

SOC 2 Type II, ISO 27001 / 27701 / 42001, GDPR, and CCPA, plus third-party audits.

Know the headlines; route the details

Answer the four above with confidence. For specific contractual, compliance, or configuration questions, point the user to the security/trust portal and loop in the right internal team. A security answer that’s subtly wrong does more damage than saying “let me get you the precise answer” — when unsure, route, don’t guess.
